## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

> メニュー表示→入力受付→処理分岐→ループの流れを記述してください。
> フローチャートまたは疑似コード（箇条書き）で記述してください。

```
1. メインメニューを表示する
2. ユーザーの入力を受け取る
3. 入力に応じて分岐する
   - 1の場合: （商品検索）
   - 2の場合: （購入商品表示）
   - 3の場合: （支払方法選択）
   - 4の場合: （合計計算）
   - 5の場合: （支払方法選択）
4. メインメニューに戻る（1へ）
```

### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。

#### 機能1: 商品検索 (F01)

```
商品名等の入力
入力内容の取得
入力内容の確認
データベースを検索
検索結果を判定
検索結果を表示
商品を選択
購入リストに追加
計算に遷移(買い物を確定)
```

#### 機能2: 合計計算(F02) 預かり金額、おつり、税抜金額 、税込金額 

```
1. 合計金額を計算する
   - 合計 = 商品1 * 個数 + 商品2 * 個数 + 商品3 * 個数 + 商品4 * 個数 + 商品5 * 個数
2. 登録されている商品の税率をCSVから検索する
3. 呼び出された税率を対応する商品にかける(*)
4.合計金額の値を支払いに渡す
```

#### 機能3: 購入商品表示(F03)

```
1. 商品番号を受け取る（F01から）
2. 商品番号に対応する商品名・単価を商品マスタから取得する
3. 該当商品が既にリストにあれば数量を+1、なければ新しく追加する
4. 現在のカートを画面に表示する
5. 次の商品番号の入力を待つ、または会計へ進む
```

#### 機能4: 支払方法 (F04)

```
1.合計金額表示(printf)
2.支払方法選択（クレカ、現金、クーポン）
3.現金なら料金の値を取得→合計金額以上になるまで料金の値を繰り返し受け取り、計算して支払完了。
4.クレカなら決済処理を行い成功なら完了、失敗ならエラー表示
5.決済完了でメインメニューに戻る
（アルゴリズムを記入）
```

### 4.3 エラーハンドリング（想定される異常系）

> ユーザーが「想定外の操作」をした場合にどうするか検討してください。

| 異常系シナリオ | 対処方針 |
| :--- | :--- |
| 一致する商品がない場合 |「商品が見つかりません」と表示、再入力ボタン表示 |
| 金額が不足している場合 |「お金を投入してください」と表示 |
| カートに商品が入っていない場合 | 「商品を入力してください」と表示|
