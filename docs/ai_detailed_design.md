## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

> メニュー表示→入力受付→処理分岐→ループの流れを記述してください。
> フローチャートまたは疑似コード（箇条書き）で記述してください。

```
1. 起動時に商品マスタCSV（products.csv）を読み込む
  - 読込成功: 商品一覧をメモリ上に展開する
  - 読込失敗: エラー表示して再試行、または終了を選択させる
2. メインメニューを表示する
3. ユーザーの入力を整数で受け取る（1/2/9）
4. 入力に応じて分岐する
   - 1の場合: 購入フローを開始する
     1) カートを初期化
     2) カテゴリ選択 -> 商品選択 -> 数量入力を繰り返し、カートに追加
     3) 税区分を選択して金額計算
     4) 支払方法を選択して支払い処理実行
     5) 取引ヘッダ/明細をCSV保存し、操作ログを追記
     6) 完了画面を表示後、メインへ戻る
   - 2の場合: 取引履歴を読み込み、一覧表示後にメインへ戻る
   - 9の場合: プログラムを終了する
   - それ以外: エラーメッセージを表示して再入力
5. メインメニューに戻る（2へ）
```

### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。

#### 機能1: 購入処理（商品選択・計算・会計）

```
対象関数: load_products, select_product, calc_total, payment

[load_products]
1. products.csvを読み込みモードで開く
2. ヘッダ行を読み飛ばす（存在する場合）
3. 各行をCSV分割し、列数と型を検証する
4. product_id, product_name, category, price_excl_tax, is_active を商品マスタ配列へ格納する
5. is_active=1 の商品のみ選択対象として有効化する
6. 読込件数を返却する

入力:
- file_path（例: products.csv）

出力:
- products[]（商品マスタ配列）
- product_count（読込件数）
- load_result（成功/失敗）

エラー処理:
- ファイルオープン失敗: エラーコード返却、起動時に再試行/終了を選択
- CSV列数不足/超過: 当該行をスキップし警告ログを残す
- 数値変換失敗（id/price/is_active）: 当該行をスキップ
- product_count=0: 購入機能を無効化してメニューへ戻す

[select_product]
1. load_productsで読み込んだ商品マスタをカテゴリ条件で絞り込み表示する
2. 商品番号入力を受け取り、該当チェックする
3. 数量入力を受け取り、1以上か検証する
4. カート明細に商品を追加する

入力:
- product_id（ユーザー入力）
- quantity（ユーザー入力）

出力:
- カート更新結果（成功/失敗）
- 選択商品情報（id, name, price, tax）

エラー処理:
- 不正な商品番号: 「該当商品なし」を表示し再入力
- 数量が0以下: 「1以上を入力」を表示し再入力
- 入力が数値でない: 入力バッファをクリアして再入力

[calc_total]
1. カート明細を1件ずつ走査する
2. 税区分に対応する税率を決定する
3. 行単位で税抜小計、税額、税込小計を計算する
4. 合計値（税抜/税額/税込）を集計して返す

入力:
- cart_items[]（商品単価、数量）
- tax_mode（IN_STORE/TAKEOUT/TAX_FREE）

出力:
- subtotal_excl_tax
- tax_amount
- total_incl_tax

エラー処理:
- カートが空: 計算せずエラーコード返却
- 税区分が不正: デフォルト税率を適用せずエラー扱い

[payment]
1. 請求金額を表示する
2. 支払方法を受け取る（現金/カード/電子マネー）
3. 現金の場合は入金額を繰り返し受け付ける
4. 不足時は不足額を再表示、充足時はお釣りを計算
5. 決済完了を返却する

入力:
- total_incl_tax
- payment_method
- paid_amount（現金時）

出力:
- payment_result（成功/失敗）
- change_amount（現金時）

エラー処理:
- 現金不足: 再入力を促す
- 不正な支払方法: メニューへ戻して再選択
- 負の金額入力: 無効として再入力
```

#### 機能2: 保存・履歴表示・操作ログ

```
対象関数: save_transaction, show_history, log_operation

[save_transaction]
1. 取引IDを採番する（日時+連番）
2. 取引ヘッダをtransactions.csvへ追記
3. 取引明細をtransaction_items.csvへ追記
4. 成功/失敗を返却する

入力:
- transaction_header
- transaction_items[]

出力:
- 保存結果（成功/失敗）

エラー処理:
- ファイルオープン失敗: エラー表示し、処理を中断
- 書き込み途中失敗: ログに失敗内容を残し再試行案内

[show_history]
1. transactions.csvを読み込む
2. 必要件数（最新N件）を抽出する
3. 画面表示形式に整形して出力する

入力:
- 表示条件（件数、期間、取引ID）

出力:
- 取引履歴一覧

エラー処理:
- ファイルなし: 「履歴なし」を表示
- CSV破損行: スキップして警告表示

[log_operation]
1. 操作イベントを受け取る
2. timestamp, transaction_id, action, detail を生成
3. operation_log.csvに1行追記する

入力:
- action
- detail
- transaction_id（任意）

出力:
- なし（副作用: ログ追記）

エラー処理:
- ログファイル書き込み失敗: 標準エラーへ出力し本処理は継続
```

### 4.3 エラーハンドリング（想定される異常系）

> ユーザーが「想定外の操作」をした場合にどうするか検討してください。

| 異常系シナリオ | 対処方針 |
| :--- | :--- |
| 数字以外の文字が入力された場合 | 入力バッファをクリアし、エラーメッセージを表示して再入力を促す |
| 金額が不足している場合 | 不足額を表示し、追加入金ループを継続する |
| 存在しない商品IDを入力した場合 | 商品一覧を再表示し、再選択させる |
| 数量に0以下を入力した場合 | 1以上の入力を求める |
| 商品マスタCSVが開けない場合 | 起動時にエラー表示し、再試行または終了を選択させる |
| 商品マスタCSVに不正行がある場合 | 不正行のみ読み飛ばし、正常行で処理継続する |
| CSVファイルが開けない場合 | エラーを表示し、保存系処理は中断。購入処理自体は安全に終了させる |
| 税区分/支払方法の選択肢外入力 | メニューを再表示して再選択させる |
| 履歴CSVに不正行がある場合 | 不正行を読み飛ばし、正常行のみ表示する |

### 4.4 キャンセル処理フロー

> 各画面でのキャンセル動作を定義する。キャンセルキーは全画面共通で **「0」** を使用する。

#### 画面別キャンセル動作

| 画面名 | キャンセル時の動作 | カートへの影響 |
| :--- | :--- | :--- |
| カテゴリ選択画面 | メインメニューへ戻る | カートをクリアする |
| 商品一覧/検索画面 | カテゴリ選択画面へ戻る | カートは保持する |
| 数量入力画面 | 商品一覧/検索画面へ戻る | 選択中の商品は追加しない（カートは保持） |
| カート確認画面 | 購入全体をキャンセルしてメインメニューへ戻る | カートをクリアする（操作ログに記録） |
| 税区分選択画面 | カート確認画面へ戻る | カートは保持する |
| 支払方法選択画面 | 税区分選択画面へ戻る | カートは保持する |
| 支払い処理画面（現金） | 現金投入前のみキャンセル可。支払方法選択画面へ戻る | カートは保持する |
| 履歴画面 | メインメニューへ戻る | 影響なし |

> ※ 支払い処理画面で現金を一度でも投入した後はキャンセル不可とする（投入済み金額の扱いが未確定なため）。

#### キャンセル処理アルゴリズム（cancel_purchase）

```
対象関数: cancel_purchase

[cancel_purchase]
1. カート内の全明細を破棄する（cart_count = 0 にリセット）
2. 税区分・支払方法の選択状態をリセットする
3. 操作ログに CANCEL_PURCHASE アクションを記録する
   - action: CANCEL_PURCHASE
   - detail: items_in_cart=<キャンセル時点のカート件数>
   - transaction_id: 未発行のため空文字
4. メインメニューへ戻る

入力:
- cart（現在のカート情報）
- items_in_cart_count（ログ記録用）

出力:
- なし（副作用: カートクリア・ログ追記）

エラー処理:
- ログ書き込み失敗: 標準エラーへ出力し、カートクリアとメニュー遷移は継続する
```

#### カート確認画面での商品削除アルゴリズム（delete_cart_item）

```
対象関数: delete_cart_item

[delete_cart_item]
1. カート確認画面で「d」を入力すると削除モードへ遷移する
2. カート内の明細行番号（1〜cart_count）を入力させる
3. 入力値の範囲チェックを行う
4. 該当行を配列から削除し、後続行を詰める
5. カート確認画面を再表示する

入力:
- line_no（削除する行番号、ユーザー入力）

出力:
- 削除結果（成功/失敗）

エラー処理:
- 範囲外の行番号: 「無効な番号です」を表示して再入力
- カートが空の状態で削除操作: 「カートに商品がありません」を表示
- 入力が数値でない: 入力バッファをクリアして再入力
```

## 5. 備考・メモ

> 設計中に気になった点や、明日の実装に向けた注意点があれば記録してください。

* 金額計算はfloat依存を減らし、実装時は「円単位の整数」で統一する（税額計算時のみ丸め規則を適用）。
* 取引IDは重複防止のため、日時 + 連番で採番する。
* CSVヘッダ行の有無を統一し、初回作成時のみヘッダを書き込む。
* 商品数が増えるため、将来的にカテゴリ + 部分一致検索を標準機能にする。
* 例外時でも操作ログだけは可能な範囲で残し、原因追跡をしやすくする。
