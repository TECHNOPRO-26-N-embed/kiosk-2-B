## 3. 基本設計

### 3.1 画面遷移図

> テキスト形式で画面の状態遷移を記述してください。
> 例: `メインメニュー → [1] → 購入画面 → [確定] → 完了画面 → メインメニュー`

```
起動画面
→ メインメニュー
→ [1: 購入開始]
→ カテゴリ選択画面
→ [カテゴリ選択]
→ 商品一覧/検索画面
→ [商品選択]
→ 数量入力画面
→ [追加]
→ カート確認画面
→ [商品追加を続ける] → カテゴリ選択画面
→ [会計へ進む] → 税区分選択（店内/持ち帰り/免税）
→ 支払方法選択画面（現金/カード/電子マネー）
→ 支払い処理画面
→ [支払い完了]
→ 完了画面（合計・お釣り・取引ID表示）
→ メインメニュー

メインメニュー
→ [2: 取引履歴表示] → 履歴画面 → メインメニュー

メインメニュー
→ [9: 終了] → 終了

```

### 3.2 データの洗い出し

> このシステムが**覚えておくべき情報**を洗い出してください。
> まずは「何の情報が必要か」を自然言語で書き出せばOKです。
> （C言語での型や構造体の定義は、明日の詳細設計で行います）

| 情報の名前 | 内容・説明 | 例 |
| :--- | :--- | :--- |
| 商品マスタ情報 | 商品ID、商品名、カテゴリ、税抜単価、有効フラグ。CSVから読み込んで更新可能にする。|ID: 101, 商品名: おにぎり, カテゴリ: 食品, 単価: 120|
| カート情報 | 現在の購入明細。商品ID、商品名、単価、数量、小計を保持。 | おにぎり, 120円, 数量2, 小計240 |
| 取引ヘッダ情報 | 1回の会計単位の情報。取引ID、日時、税区分、支払方法、税抜合計、税額、税込合計、預かり金、お釣り | TX20260519001, 2026-05-19 14:05:10, 店内, 現金, 1000, 100, 1100, 1200, 100 |
| 取引明細情報 |取引に含まれる商品ごとの明細。取引IDと紐づける | TX20260519001, 101, おにぎり, 120, 2, 240 |
| 操作ログ情報 | ユーザー操作の履歴。何をいつ実行したかを保存 | 2026-05-19 14:03:01, SELECT_ITEM, 商品ID=101 |
| 税設定情報 | 店内税率、持ち帰り税率、免税率、端数処理ルール | 店内10%, 持ち帰り8%, 免税0%, 四捨五入 |

### 3.3 CSV保存形式

> CSVファイルにどのような形式でデータを保存するか定義してください。

```
1) 商品マスタCSV（products.csv）
product_id,product_name,category,price_excl_tax,is_active
101,おにぎり,食品,120,1
102,緑茶,飲料,150,1

2) 取引ヘッダCSV（transactions.csv）
transaction_id,timestamp,tax_mode,payment_method,subtotal_excl_tax,tax_amount,total_incl_tax,paid_amount,change_amount
TX20260519001,2026-05-19 14:05:10,IN_STORE,CASH,1000,100,1100,1200,100

3) 取引明細CSV（transaction_items.csv）
transaction_id,line_no,product_id,product_name,unit_price_excl_tax,quantity,line_subtotal_excl_tax,line_tax_amount,line_total_incl_tax
TX20260519001,1,101,おにぎり,120,2,240,24,264
TX20260519001,2,102,緑茶,150,1,150,15,165

4) 操作ログCSV（operation_log.csv）
timestamp,transaction_id,action,detail
2026-05-19 14:03:01,TX20260519001,SELECT_CATEGORY,食品
2026-05-19 14:03:15,TX20260519001,ADD_ITEM,product_id=101 quantity=2
2026-05-19 14:05:10,TX20260519001,PAYMENT_COMPLETED,method=CASH total=1100

```
