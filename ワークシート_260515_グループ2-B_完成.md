## 3. 基本設計

### 3.1 画面遷移図

> テキスト形式で画面の状態遷移を記述してください。
> 例: `メインメニュー → [1] → 購入画面 → [確定] → 完了画面 → メインメニュー`

```
メインメニュー →[1]　 → 商品検索表示 → [カテゴリ選択] → 商品選択 → 個数入力 → 確認画面 → [確定] → 支払方法選択画面表示 → 支払い → 支払い完了画面表示 → メインメニュー

↑　常にメインメニューに戻るボタンがある
↑エラー出たら、「メインメニューに戻る」と表示
↑決済エラー出たら、支払方法選択画面再表示

```

### 3.2 データの洗い出し

> このシステムが**覚えておくべき情報**を洗い出してください。
> まずは「何の情報が必要か」を自然言語で書き出せばOKです。
> （C言語での型や構造体の定義は、明日の詳細設計で行います）

| 情報の名前 | 内容・説明 | 例 |
| :--- | :--- | :--- |
| 商品情報|商品番号、商品名、単価|0001、アイス、300|
| 購入数|1回の購入数|3個|
|料金 |合計、税率 |商品合計、8%(食品類) |
|支払い情報 |現金、クレジット、クーポン |現金、10％|
